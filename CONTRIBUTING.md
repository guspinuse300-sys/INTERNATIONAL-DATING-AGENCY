# Contributing to International Dating Agency

Thank you for considering contributing to the International Dating Agency project! We welcome your contributions and feedback. Please follow these guidelines to ensure a smooth collaboration.

## Contribution Guidelines

1. **Issues**: Before contributing, please check our open issues to see if your idea has already been suggested or is being worked on. If not, feel free to create a new issue!
2. **Fork the Repository**: Fork the repository to your GitHub account. 

## Branch Naming Conventions

We follow a specific branch naming convention to ensure clarity in our workflow:

- Use `feature/<description>` for new features (e.g., `feature/add-user-auth`).
- Use `bugfix/<description>` for bug fixes (e.g., `bugfix/fix-login-issue`).
- Use `hotfix/<description>` for emergency fixes (e.g., `hotfix/urgent-security-patch`).
- Use `chore/<description>` for routine tasks that don't fit the other categories (e.g., `chore/update-dependencies`).

## Commit Message Standards

When committing changes, please adhere to the following commit message format:

- Use the present tense (e.g., "Add feature" instead of "Added feature").
- Begin with a capitalized word and do not end with a period.
- Optionally, include a detailed description of your changes after the summary line, separated by a blank line.

**Example:**

```
Add user authentication feature

Implemented OAuth2 login functionality.
```

## Pull Request Process

1. **Create a Pull Request**: After pushing your changes, navigate to the original repository and create a pull request.
2. **Provide Context**: Clearly describe the changes you made and the issue your pull request addresses.
3. **Review**: Your pull request will be reviewed by the project maintainers. Be open to feedback and make necessary adjustments.
4. **Merge**: Once approved, your pull request will be merged into the main branch.

Thank you for your contributions! Together, we can make this project better.
